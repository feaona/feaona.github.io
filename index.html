02 · Problem Definition

问题定义与洞察

Target User · C 端
张伟 / 小张
C 端体验者 · 活动发起人
碎片化时间 情绪价值 拒垫资
"接了组织部门团建的活，期望在不影响本职工作、且绝不自己垫资的前提下，快速搞定让大家满意的方案。"
Pain 01

资金高压：业财断裂与垫资困境

活动发起人面临个人垫付大额资金的巨大压力，传统对公流程冗长极易耽误预订，线下履约导致业财数据断裂产生合规盲区。

Pain 02

高昂成本：方案同质化与决策内耗

工业化模板难满足 Z 世代，需求填写繁琐，统一全员意见的协调成本极高，决策周期无限拉长，严重消耗发起人精力。

Pain 03

体验割裂：行后发票焦虑与分享受阻

活动结束后常因丢失纸质发票产生焦虑，优质照片分散难以收集，阻碍员工发朋友圈分享、获取情绪价值的核心诉求。

Target User · C 端
陈宇 / 宇哥
C 端体验者 · 情绪价值捕手
拒绝走马观花 沉浸体验 社交分享
"平时上班已经够累了，出去玩千万别搞得像个换地方军训。我只想要真正的沉浸式体验，最好能让我拍出大片、发朋友圈出圈的惊喜瞬间，这就是「有效团建」。"
Pain 01

团建如军训，沉浸感全无

行程千篇一律、形式大于体验，工业化模板无法满足 Z 世代对个性化与深度沉浸的渴望，团建沦为占用周末的负担。

Pain 02

精彩瞬间稍纵即逝，难以留存

优质照片散落各处、收集整理成本极高，拍不出"大片感"的场景设计让人无从下手，错失制造朋友圈爆款的最佳时机。

Pain 03

情绪出口缺失，社交分享受阻

没有配套的社区与晒单场景，情绪价值无处释放；缺乏能引发共鸣的内容承载，让参与者失去主动分享与二次传播的动力。

行中提供服务日历与管家群聊。行后打造云相册与晒单广场，聚合影像，鼓励社交分享，形成用户自传播闭环。

核心架构策略

双场景分流

首页仅保留行程通知与运营展示，个人深度管理分割至独立大厅，精准兼顾"聚焦泛浏览"与"强任务"双重场景。

核心业务前置

将 AI 助手设置一级核心 Tab，以"对话式定制"取代繁琐表单，实现核心服务一键直达，大幅提升需求匹配效率。

职场社交与内容入口

社区前沿"同事圈"与"精选评价"，依托熟人社交与真实内容构建信任背书，有效推动用户粘性与裂变增长。

层级深度控制

核心高频操作（如行程查看、AI 对话定制、同事圈互动）一级直达，辅助功能严格收敛于二级。

闭环设计

"行程"独立为核心管理模块，串联行前（合同）、行中（管家）、行后（相册），构建服务生命周期完整闭环。

可扩展性

在 C 端完整体验架构下，打造了"管家群"、"合同/发票"、"参与人员"等功能节点，为 B 端数据打通预留充足架构空间。

09 · Visual Identity

设计原则与情绪板

从「商务属性」与「智能效率」的双重痛点出发，构建兼具极简呼吸感与专业信任的视觉语言。

Breathable 透气

去除大面积色块，使用 90% 的留白与极浅底色。让高品质旅游风光图和 IP 成为主角，消除传统 App 的拥挤感，减轻商务用户的认知压力。

Professional 专业

正文采用高对比度的「深空蓝灰」，克制色彩的使用。平衡 IP 的可爱感，确保作为商旅工具的高级感、严谨性与权威信任感。

Focused 聚焦

提取高饱和度的「鸭嘴橙」作为绝对转化色。在极素雅的背景下，形成锋利的视觉焦点，配合简洁卡片引导用户一键预订快速决策。

01 · Project Background

为何重构？

随着硬件制造向轻薄化、定制化和柔性生产转型，企业订单模式正发生剧变。前期调研发现，原有的通用型 CRM 在业务链路上存在严重的「水土不服」——系统仅停留在「销售人脉流转」，一旦商机进入打样、核价、试产等硬件特有环节，系统便完全失去掌控力。

3–5 天
非标核价极度黑盒化
销售依赖微信和 Excel 跨部门沟通图纸与 BOM，极易漏算模具费，非标定制件平均报价周期长达 3–5 天，大量商机因此流失。
28 %
样品流转失控，客诉率高
样品在车间对前端销售透明度为零，多次打样（V1–V4）导致需求变更与测试报告张冠李戴，样品延期引发的客诉率高达 28%。
>40 %
数据割裂，排产预测失真
人工解析客户非标 Forecast 表格并手动录入 ERP，错误率高，供应链备料偏离度超过 40%，前端不知道卖啥，后端盲目备货。

01 · Design Objectives

设计目标 | Design Objectives

破除孤岛，全景可视化
End-to-End Visibility

传统通用型 CRM 让各部门处于「信息烟囱」。通过界面串联销售、研发与供应链，借助 Stage-Gate 视图具象化跨部门业务流程（如样品全生命周期推进器），消除跨部门沟通摩擦。

化繁为简，认知降噪
Cognitive Reduction

制造业 B2B 系统伴随海量生硬数据与复杂决策链。将干瘪数据转化为直觉化图形——360° 客户雷达图与 MCR 大客户权力地图，将隐性的利益关系显性化，大幅降低用户的认知负担。

所见即所得的沉浸体验
Immersive Interaction

引入「无限画布」与「拖拽式」交互范式。CPQ 报价配置采用左右分屏 + 骨架屏加载的实时联动逻辑，配置改动即时反馈，赋予用户极强的操作控制感与安全感。

极简克制的 SaaS 质感
Professional Aesthetics

秉持极简与克制，舍弃多余装饰。大面积留白与清晰的信息层级规范，严格对齐高级别 UI 体系，用合理字阶与品牌高亮色引导焦点，让沉重的 B2B 工业软件也传达出现代智造基因。

User Pain Points

Design Opportunities

智能CPQ与协同看板：引入3D预览组件；将核价拆解为可视化SLA节点，系统自动计算阶梯报价与模具分摊。

STEP 03
打样与验证

User Pain Points

进度失控与版本混乱：打样在车间流转对销售完全黑盒；样品多次修改后，变更和测试反馈难以对应。

Design Opportunities

打样物流化与版本树：打造打样进度大盘（如物流般透明）；建立样品版本树视图，强制关联需求与测试报告。

STEP 04
定点与量产

电子签批流组件

User Pain Points

合规风险与确权慢：纸质承认书跨国流转慢；缺乏系统级"最终配置快照"，后续量产极易发生扯皮。

Design Opportunities

电子契约与数字基线：集成电子签认证加速在线确权；定点瞬间系统自动锁定工艺路线，生成只读量产基线。

STEP 05
订单转化

核心接触页面
Forecast解析面板
交付协同看板
User Pain Points

数据割裂与拆单繁琐：客户滚动预测多为非标表格，销售需耗费巨量精力手动拆分录入ERP，出错率极高。

Design Opportunities

智能解析与业财一体：集成大模型解析插件秒抓Excel订单数据；打通网关一键抛转PO并实时回拉交期预警。
